# Руководство пользователя Zypo Mesh

Добро пожаловать в децентрализованную сеть Zypo! Это руководство предназначено для обычных пользователей и разработчиков. Здесь вы узнаете, как установить клиент и браузер, а также как бесплатно и анонимно разместить свой сайт в сети `.mesh`.

---

## 1. Запуск клиента и браузера (Для обычных пользователей)

Чтобы начать путешествовать по скрытым сайтам в сети Zypo, вам понадобятся две вещи: **Клиентская нода** (которая держит P2P-соединение с сетью) и **Zypo Browser** (визуальный интерфейс).

### Шаг 1: Запуск клиентской ноды
1. Убедитесь, что у вас установлен **Go** (версия 1.22+).
2. Откройте терминал в папке проекта и запустите клиент:
   ```bash
   cd dot-zypo-client
   go run main.go
   ```
   *(Опционально) Если вы хотите явно подключиться к основной координационной ноде Zypo, запустите:*
   ```bash
   go run main.go -bootstrap /ip4/213.171.27.234/tcp/8900/p2p/12D3KooWPyuxuDGwsnBB6sKvCarjfRimWhNvvXaAhYnGkesBFbEf
   ```
   *Вы увидите логи подключения. Как только появится сообщение о том, что сеть "Synced" (Синхронизировано), клиент готов к работе!*

### Шаг 2: Запуск Zypo Browser
Не закрывая первый терминал, откройте новое окно терминала:
1. Убедитесь, что у вас установлен **Node.js** (версия 18+).
2. Запустите браузер:
   ```bash
   cd dot-zypo-browser
   npm install
   npm start
   ```
Откроется красивое окно браузера Zypo. Введите любой адрес (например, `zypo://domains.zypo`) в адресную строку и наслаждайтесь децентрализованным веб-серфингом!

## 2. Размещение статического сайта (HTML/CSS/JS)

Вы можете бесплатно захостить свой сайт прямо со своего компьютера. Сеть Zypo автоматически перенаправит запросы других пользователей к вашей запущенной клиентской ноде.

### Как создать статический сайт в Zypo:
1. В корневой папке, откуда вы запускаете `dot-zypo-client` (обычно это папка `dot-zypo-client`), создайте папку `zypo_sites`.
2. Внутри `zypo_sites` создайте папку с именем вашего домена, например `myblog.mesh`.
3. Положите внутрь этой папки ваш файл `index.html` и другие ресурсы (картинки, CSS).

**Пример структуры файлов:**
```text
dot-zypo-client/
└── zypo_sites/
    └── myblog.mesh/
        ├── index.html
        ├── style.css
        └── logo.png
```

Запустите `dot-zypo-client`. Ваш сайт сразу же станет доступен всем пользователям сети по адресу `zypo://myblog.mesh` (при условии, что ваша нода онлайн)!

> **Примечание по доменам:** Чтобы закрепить домен за вашей нодой, вы можете зарегистрировать его через смарт-контракт `zypo://domains.zypo` в браузере, привязав к вашему Peer ID.

## 3. Проксирование динамических сайтов (Python, Node.js, PHP)

Если ваш сайт работает на бэкенде (например, **FastAPI, Flask, Django, Express.js**), вы не можете просто положить файлы в папку. Вместо этого Zypo предлагает встроенную функцию **Reverse Proxy (Обратный прокси)**. Zypo будет принимать запросы из децентрализованной сети и прозрачно перенаправлять их на ваш локальный сервер!

### Как настроить проксирование (Reverse Proxy):

1. **Запустите ваш сервер.**
   Например, вы написали крутой API на Python (FastAPI) или подняли веб-приложение на Flask, и оно работает локально на порту `8000`:
   ```text
   http://127.0.0.1:8000
   ```

2. **Настройте Zypo для перенаправления трафика:**
   В папке `zypo_sites` создайте специальный файл с названием `reverse_proxy.json`.
   Впишите туда соответствие между желаемым доменом в сети Zypo и вашим локальным сервером:
   ```json
   {
       "myapp.mesh": "http://127.0.0.1:8000",
       "api.myproject.zypo": "http://localhost:3000"
   }
   ```
   *Вы можете указать сразу несколько сайтов в одном файле!*

3. **Готово!**
   При запущенном клиенте (`dot-zypo-client`), любой пользователь Zypo, введя в браузере `zypo://myapp.mesh`, на самом деле обратится к вашему Python-серверу на `127.0.0.1:8000`. При этом ваш локальный сервер будет безопасно скрыт от обычного интернета.

**Обратите внимание:** Настройки в файле `reverse_proxy.json` применяются автоматически в реальном времени при сохранении файла. Перезапускать Zypo Client не нужно!

## 4. Для Разработчиков и Администраторов

Если вы хотите развернуть собственный Command Center, VPN-сервер, или вам нужны детали протокола безопасности (обход DPI, сертификаты TLS MITM, закрытые порты), **не ищите эту информацию здесь**.
