# 🌐 Zypo Mesh Network

**Zypo** — это пуленепробиваемый, децентрализованный P2P-протокол следующего поколения, объединяющий в себе зашифрованную сеть передачи данных, распределенную DNS-систему, автономную P2P-экономику на основе Gossip и неуязвимый VPN.

## 🚀 Архитектура и Основные компоненты

Система состоит из нескольких независимых узлов, которые общаются друг с другом по протоколу LibP2P:

1. **`dot-zypo-client` (Клиентский Демон):** Работает на машине пользователя. Предоставляет локальный SOCKS5, HTTP Forward Proxy и системный TUN-интерфейс для маршрутизации трафика в сеть Zypo.
2. **`dot-zypo-browser` (GUI):** Пользовательский интерфейс (на базе Electron), который взаимодействует с демоном через протокол `zypo://`.
3. **`dot-zypo-control` (Command Center):** Опциональный координационный узел. Служит как DNS-Оракул для подтверждения официальных зон (например, `.zypo`), а также выполняет роль публичного Relay-хаба для помощи узлам, находящимся за жестким NAT (Circuit Relay V2).
4. **`dot-zypo-yan` (Search Crawler & Exit Node):** Поисковый движок для скрытой сети, а также может выступать в роли VPN-провайдера (Exit Node), пропуская через себя зашифрованный трафик.

## 💰 Децентрализованная P2P Экономика (ZPCN)

Вся внутренняя экономика (ZPCN) работает **без центральных серверов** и смарт-контрактов:
* **Gossip Protocol:** Транзакции транслируются в сеть через алгоритм сплетен (gossip-protocol). Узлы пересылают транзакции друг другу, обеспечивая атомарную дедупликацию для предотвращения "сетевых штормов".
* **Ed25519 Подписи:** Каждая транзакция криптографически подписывается отправителем. Баланс хранится и проверяется локально на каждой ноде.
* **Prepaid Traffic:** Оплата за VPN и сервисы работает по принципу предоплаты. Клиент отправляет транзакцию провайдеру, провайдер валидирует её в P2P-сети и выделяет пользователю квоту трафика (в байтах). Когда квота заканчивается, соединение прерывается до следующей транзакции.

## 🛡️ Bullet-Proof VPN & Направление Трафика

Zypo предоставляет мощный инструмент обхода блокировок:
* **Мультиплексирование и Anti-DPI:** Zypo динамически переключается между TCP, QUIC (UDP) и WebSocket-over-TLS. Трафик маскируется под обычный HTTPS (порт 443/8901), чтобы обходить Deep Packet Inspection и жесткие фаерволы.
* **Double Tunneling (Двойное туннелирование / Circuit Relay):** Если целевой VPN-провайдер недоступен напрямую из-за NAT, LibP2P автоматически использует Command Center как транзитный Relay-хаб. 
* **Bridge Protocol:** Узлы с открытыми портами могут автоматически становиться мостами (Bridges) для ретрансляции трафика пользователей за строгим NAT.
* **TLS MITM Proxy:** Встроенный локальный HTTPS Forward Proxy позволяет браузерам прозрачно открывать `.zypo` сайты с генерацией сертификатов на лету.
* **UDP Datagram Forwarding:** Полная поддержка туннелирования TCP и UDP трафика, включая DNS-запросы (через порт 53).

## 🔌 Сетевые Порты Zypo

Демон Zypo использует следующие порты:
* **8900** — TCP/QUIC: Основной P2P-порт для прослушивания соединений (Command Center, Server Node).
* **8901** — TCP/WS: Резервный порт для WebSocket-over-TLS (маскировка под HTTPS).
* **8903** — HTTP: Локальный RPC-сервер (взаимодействие Zypo Browser и демона).
* **8999** — UDP: Локальный поиск узлов в LAN.
* **1080** — TCP: Локальный SOCKS5-прокси (когда VPN активен).
* **53**   — UDP: Перехват и туннелирование DNS-запросов (когда VPN активен).

## 🌍 Распределенная DNS & Безопасный Хостинг

Сайты внутри Zypo физически располагаются на компьютерах самих пользователей.
* **DNS Resolution:** Поиск IP-адресов происходит не через классические DNS серверы, а через распределенную хэш-таблицу (Kademlia DHT). 
* **Official Zones vs Mesh Zones:** Официальные зоны (например, `.zypo`) требуют регистрации через Command Center для защиты от сквоттинга. Открытые зоны (например, `.mesh`) полностью децентрализованы и не требуют ничьего разрешения — ваше Peer ID и есть гарантия владения.
* **Хостинг:** Чтобы разместить сайт, достаточно положить статические файлы в директорию `zypo_sites/твой-домен.zypo/`. Нода будет автоматически раздавать их P2P-пирам по запросу.

## 🛠️ Быстрый запуск

### 1. Клиентская нода (Client)
Для подключения к сети вам потребуется адрес любой доверенной Bootstrap-ноды. По умолчанию клиент попытается использовать встроенные адреса:
```bash
go run dot-zypo-client/main.go
```
Если вы хотите явно указать основную координационную ноду:
```bash
go run dot-zypo-client/main.go -bootstrap /ip4/213.171.27.234/tcp/8900/p2p/12D3KooWPyuxuDGwsnBB6sKvCarjfRimWhNvvXaAhYnGkesBFbEf
```

### 2. Запуск Браузера
После того, как клиентская нода запустится и перейдет в состояние `[DHT] Ready`, откройте Zypo Browser:
```bash
cd dot-zypo-browser
npm install
npm start
```

### 3. VPN Провайдер (Exit Node)
Если вы хотите зарабатывать `ZPCN`, раздавая интернет:
```bash
cd dot-zypo-yan
go build -o yan
./yan -vpn -port 9000
```
*(Убедитесь, что порты 9000 TCP и UDP открыты в вашем фаерволе!)*
